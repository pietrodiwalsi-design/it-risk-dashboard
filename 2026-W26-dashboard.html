2 AI Gateway Authentication Bypass Exposes API Credentials 100/100
📂 Vulnerability Management 📡 NVD

Als we niets doen: Attackers will steal OpenAI and third-party API credentials, enabling unauthorized AI service abuse, data exfiltration, and injection attacks against downstream systems.

Impact Level Detail
💰 Financial 🔴 Critical Compromised API credentials enable fraudulent AI transactions, unauthorized data access, and potential regulatory fines for failing to protect customer data. Exposure of financial data through AI systems could trigger significant breach notification costs and reputational damage with customers and investors.
🏢 Organisational 🔴 Critical Attackers can inject malicious prompts into AI-driven customer service, underwriting, or claims systems, corrupting decision outputs and operational integrity. Service disruption or compromise of AI-assisted processes directly impacts claims processing, policy issuance, and customer interactions.
💻 IT 🔴 Critical This vulnerability is actively exploitable with no mitigation available below version 1.84.0 and must be patched immediately. Exploitation requires zero additional sophistication and leaves comprehensive audit trails of unauthorized API usage.
✅ Recommended action: Audit all LiteLLM deployments for versions below 1.84.0 immediately; prioritize upgrade to 1.84.0+ within 48 hours and revoke all API credentials that passed through vulnerable instances.
3 Message Processing Library Accepts Untrusted Data Without Validation 100/100
📂 Vulnerability Management 📡 NVD

What's happening? Our ASP.NET applications using an older version of MessagePack are configured to accept external data without the security controls designed to prevent malicious input attacks.

Als we niets doen: Attackers can send specially crafted requests that consume excessive server resources, causing service degradation or outage for our customers and policy holders.

Impact Level Detail
💰 Financial 🔴 Critical An extended outage could trigger SLA breaches, policyholder refund claims, regulatory fines for service disruption, and reputational damage in a market where availability and trust are core differentiators. Fraud claims tied to service failures could increase significantly.
🏢 Organisational 🔴 Critical The vulnerability enables denial-of-service attacks that can knock critical policy processing, claims systems, and customer portals offline without warning. Recovery requires incident response activation and potential manual transaction processing.
💻 IT 🔴 Critical This library flaw is actively exploitable and requires immediate patching to version 2.5.301 or 3.1.7 across all affected applications. The default insecure configuration means no code changes are needed by attackers—only network access to affected endpoints.
✅ Recommended action: Immediately inventory all ASP.NET applications using MessagePack, prioritize patching within 48 hours, and enable monitoring for suspicious serialization requests while patches are deployed.
